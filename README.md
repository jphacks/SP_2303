## 製品概要
### 背景(製品開発のきっかけ、課題等）
味覚園(地元の焼肉屋)に行こう行こうと思っていたら、すっかり忘れて2年経ってしまった。</br>
おまつ(地元の居酒屋)に久々に来店したら、前回どれが美味しかったのかを忘れてしまった。写真アプリを必死にスクロールした。</br>
このような経験から、「あそこ行きたい！」「これ美味しかった！」を記録できる、自分だけのご飯マップアプリを作ろうと考えた。

#### 課題と原因

① 外食で、新しい店に挑戦したいが、結局いつも同じような店に足を運んでしまう。</br>
- 行きたいと思っていたお店を忘れてしまうから。（飲食店名を聞けば、そういえば行きたい！という気持ちになる）
- 新しい店を見つけるきっかけがないから。（何か後押しするものがあれば行ってみたい）

② 自分が過去に食べて美味しかった料理を見たい時に、写真を振り返るのは大変。</br>
- 写真アプリでは、大量の写真の中からスクロールする必要があり、その上どれが美味しかったかのかもわからないから。
- レビューサービスは、自分の感想を自分だけが閲覧する為には作られていないから。
</br>

これらの課題を解決するために、
- 行ってみたいお店、行ったお店、食べたご飯を見返しやすい地図形式で記録できる機能
- 新しい飲食店を見つけるきっかけとなるレコメンド機能 </br>

を含むアプリケーションを開発したいと考えた。

### 製品説明（具体的な製品の説明）

「外食をより楽しく、便利に！」をコンセプトとした 外食 × Tech のモバイルアプリケーション
</br>

自分が訪れた飲食店や、行ってみたい飲食店を地図上に登録できる。
登録した飲食店には、写真、コメント、評価を記録し、いつでも振り返ることができる。

ユーザーにおすすめの店を画像でレコメンドすることで、新しい飲食店を見つけるきっかけを作る。
レコメンドはユーザーの位置情報などの情報を元に決定され、他のユーザーが記録した写真が使用される。

### 特長
#### 1. ご飯マップ
このアプリケーションのメインの画面。行ってみたいお店、行ったお店、食べた食事の記録と閲覧を行う。

#### 2. Swipe UI
おすすめのお店を画像でレコメンドする画面。
これまでは「お店を探す」という作業は文字ベースで展開されてきた。GoogleMapなどではお店の名前で検索したり、地図上で店名をタップしてからでないとどんな料理かわからない。その点に注目した私たちは、マッチングアプリのように、気になる飲食店の画像からお店を探せるUIを考案、実装した。

#### 3. キャラクター育成機能
Swipe UIの実現のためには、多くのユーザーにたくさんの記録を公開をしてもらうことが必要になることから、記録してもらえるように更なるモチベーションを用意する必要があり、育成機能を実装した。キャラクターの画像には3DCGと特殊なテクスチャを用いており、アニメ調の見た目でありながら柔らかさを表現している。

### 解決出来ること
行ってみたい飲食店をマップ画面に強調表示したり、レコメンドされた飲食店を行ってみたい飲食店として登録できる。この機能によって、行ってみたい飲食店への訪問を促したり、行ってみたい飲食店を見つけるきっかけを生むため、同じような店に足を運んでしまう問題が解消され、今まで知らなかった食の発見が期待される。

自分だけのご飯マップとして、過去に行った飲食店の食事を容易に振り返ることができる。
